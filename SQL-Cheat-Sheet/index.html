GTU SQL Complete Cheat Sheet

Computer Engineering - Quick Revision Guide

Database Operations Commands to create, manage, and delete databases
Create Database
CREATE DATABASE database_name;
Creates a new database
Use Database
USE database_name;
Selects the database to work with
Show Databases
SHOW DATABASES;
Lists all available databases
Drop Database
DROP DATABASE database_name;
Permanently deletes a database
Warning: This action cannot be undone!
Table Operations Commands to create, modify, and manage database tables
Create Table
CREATE TABLE students ( student_id INT PRIMARY KEY AUTO_INCREMENT, name VARCHAR(50) NOT NULL, age INT, grade CHAR(1), join_date DATE );
Creates a new table with specified columns
Show Tables
SHOW TABLES;
Lists all tables in current database
Describe Table
DESCRIBE table_name;
Shows the structure of a table
Alter Table (Add Column)
ALTER TABLE students ADD COLUMN email VARCHAR(100);
Adds a new column to existing table
Alter Table (Modify Column)
ALTER TABLE students MODIFY COLUMN name VARCHAR(100);
Changes column definition
Alter Table (Drop Column)
ALTER TABLE students DROP COLUMN email;
Removes a column from table
Rename Table
ALTER TABLE old_name RENAME TO new_name;
Changes the name of a table
Drop Table
DROP TABLE table_name;
Permanently deletes a table
Truncate Table
TRUNCATE TABLE table_name;
Removes all data but keeps table structure
Data Manipulation Commands to insert, update, and delete data in tables
Insert Single Record
INSERT INTO students (name, age, grade) VALUES ('Rahul', 20, 'A');
Adds a single row to the table
Insert Multiple Records
INSERT INTO students (name, age, grade) VALUES ('Priya', 21, 'B'), ('Amit', 19, 'A'), ('Neha', 20, 'C');
Adds multiple rows in single query
Insert From Select
INSERT INTO top_students (name, grade) SELECT name, grade FROM students WHERE grade = 'A';
Inserts data from another table
Update Records
UPDATE students SET grade = 'A+' WHERE student_id = 101;
Modifies existing data
Update Multiple Columns
UPDATE students SET grade = 'A+', age = age + 1 WHERE student_id = 101;
Updates multiple fields at once
Delete Records
DELETE FROM students WHERE age > 25;
Removes specific rows
Basic Queries Fundamental SELECT statements to retrieve data
Select All Columns
SELECT * FROM students;
Retrieves all columns from table
Select Specific Columns
SELECT name, grade FROM students;
Retrieves only specified columns
Where Clause
SELECT * FROM students WHERE grade = 'A';
Filters rows based on condition
Distinct Values
SELECT DISTINCT grade FROM students;
Returns unique values only
Order By (Ascending)
SELECT * FROM students ORDER BY name;
Sorts results in ascending order
Order By (Descending)
SELECT * FROM students ORDER BY age DESC;
Sorts results in descending order
Order By Multiple Columns
SELECT * FROM students ORDER BY grade ASC, age DESC;
Sorts by multiple columns
Limit Results
SELECT * FROM students LIMIT 10;
Returns only specified number of rows
Limit with Offset
SELECT * FROM students LIMIT 5 OFFSET 10;
Skips first 10 rows, returns next 5
Alias Column Names
SELECT name AS student_name, age AS student_age FROM students;
Renames columns in result set
Conditions & Operators Various operators to filter and compare data
Comparison Operators
SELECT * FROM students WHERE age >= 18;
(=, !=, <, >, <=, >=)
Logical AND
SELECT * FROM students WHERE grade = 'A' AND age > 20;
Both conditions must be true
Logical OR
SELECT * FROM students WHERE grade = 'A' OR grade = 'B';
Either condition must be true
Logical NOT
SELECT * FROM students WHERE NOT grade = 'A';
Negates a condition
IN Operator
SELECT * FROM students WHERE grade IN ('A', 'B', 'C');
Matches any value in the list
NOT IN Operator
SELECT * FROM students WHERE grade NOT IN ('D', 'F');
Excludes values in the list
BETWEEN Operator
SELECT * FROM students WHERE age BETWEEN 18 AND 22;
Range inclusive of endpoints
LIKE Operator
SELECT * FROM students WHERE name LIKE 'A%';
% for multiple chars, _ for single char
IS NULL
SELECT * FROM students WHERE grade IS NULL;
Checks for NULL values
IS NOT NULL
SELECT * FROM students WHERE grade IS NOT NULL;
Excludes NULL values
Aggregate Functions Functions that perform calculations on sets of values
COUNT
SELECT COUNT(*) FROM students;
Counts number of rows
COUNT DISTINCT
SELECT COUNT(DISTINCT grade) FROM students;
Counts unique values
SUM
SELECT SUM(age) FROM students;
Calculates sum of values
AVG
SELECT AVG(age) FROM students;
Calculates average value
MIN
SELECT MIN(age) FROM students;
Finds minimum value
MAX
SELECT MAX(age) FROM students;
Finds maximum value
GROUP_CONCAT
SELECT grade, GROUP_CONCAT(name) FROM students GROUP BY grade;
Concatenates values from multiple rows
GROUP BY & HAVING Grouping data and filtering groups
GROUP BY
SELECT grade, COUNT(*) FROM students GROUP BY grade;
Groups rows by column values
HAVING Clause
SELECT grade, AVG(age) FROM students GROUP BY grade HAVING AVG(age) > 20;
Filters groups after aggregation
GROUP BY Multiple Columns
SELECT department, grade, COUNT(*) FROM students GROUP BY department, grade;
Groups by multiple columns
ROLLUP
SELECT department, grade, COUNT(*) FROM students GROUP BY ROLLUP(department, grade);
Creates subtotals and grand total
Joins Combining data from multiple tables based on relationships
INNER JOIN
SELECT s.name, c.course_name FROM students s INNER JOIN courses c ON s.course_id = c.id;
Returns matching rows from both tables
LEFT JOIN
SELECT s.name, c.course_name FROM students s LEFT JOIN courses c ON s.course_id = c.id;
Returns all rows from left table with matches from right
RIGHT JOIN
SELECT s.name, c.course_name FROM students s RIGHT JOIN courses c ON s.course_id = c.id;
Returns all rows from right table with matches from left
FULL JOIN
SELECT s.name, c.course_name FROM students s FULL JOIN courses c ON s.course_id = c.id;
Returns all rows when match in either table
Cross Join
SELECT s.name, c.course_name FROM students s CROSS JOIN courses c;
Cartesian product of both tables
Self Join
SELECT a.name, b.name AS buddy FROM students a, students b WHERE a.buddy_id = b.student_id;
Joins table with itself
Natural Join
SELECT * FROM students NATURAL JOIN courses;
Joins tables on columns with same names
Set Operations Combine results from multiple SELECT statements
UNION
SELECT name FROM current_students UNION SELECT name FROM former_students;
Combines results, removes duplicates
UNION ALL
SELECT name FROM current_students UNION ALL SELECT name FROM former_students;
Combines results, keeps duplicates
INTERSECT
SELECT name FROM students INTERSECT SELECT name FROM scholarship_recipients;
Returns only common rows
EXCEPT (or MINUS)
SELECT name FROM students EXCEPT SELECT name FROM scholarship_recipients;
Returns rows only in first query
Subqueries Nested queries within other SQL statements
Subquery in WHERE
SELECT * FROM students WHERE age > ( SELECT AVG(age) FROM students );
Subquery returns single value
Subquery with IN
SELECT * FROM students WHERE course_id IN ( SELECT id FROM courses WHERE department = 'Computer' );
Subquery returns multiple values
EXISTS Subquery
SELECT * FROM students s WHERE EXISTS ( SELECT 1 FROM courses c WHERE c.id = s.course_id );
Checks if subquery returns any rows
Subquery in FROM
SELECT dept, avg_age FROM ( SELECT department AS dept, AVG(age) AS avg_age FROM students GROUP BY department ) AS dept_ages;
Derived table in FROM clause
Correlated Subquery
SELECT s.name FROM students s WHERE s.age > ( SELECT AVG(age) FROM students WHERE department = s.department );
Subquery references outer query
Constraints Rules enforced on data columns to maintain integrity
Primary Key
CREATE TABLE students ( student_id INT PRIMARY KEY, name VARCHAR(50) );
Uniquely identifies each record
Composite Primary Key
CREATE TABLE enrollments ( student_id INT, course_id INT, PRIMARY KEY (student_id, course_id) );
Combination of columns as primary key
Foreign Key
CREATE TABLE enrollments ( enrollment_id INT PRIMARY KEY, student_id INT, course_id INT, FOREIGN KEY (student_id) REFERENCES students(student_id), FOREIGN KEY (course_id) REFERENCES courses(course_id) );
Maintains referential integrity
Foreign Key with Actions
CREATE TABLE enrollments ( student_id INT, FOREIGN KEY (student_id) REFERENCES students(student_id) ON DELETE CASCADE ON UPDATE SET NULL );
Defines behavior when referenced data changes
NOT NULL
CREATE TABLE students ( student_id INT, name VARCHAR(50) NOT NULL );
Column cannot have NULL values
UNIQUE
CREATE TABLE students ( email VARCHAR(100) UNIQUE );
All values must be unique
CHECK
CREATE TABLE students ( age INT CHECK (age >= 18) );
Validates data against condition
DEFAULT
CREATE TABLE students ( join_date DATE DEFAULT CURRENT_DATE );
Sets default value for column
Add Constraint to Existing Table
ALTER TABLE students ADD CONSTRAINT chk_age CHECK (age >= 18);
Adds constraint after table creation
Views Virtual tables based on result sets of SQL queries
Create View
CREATE VIEW top_students AS SELECT * FROM students WHERE grade = 'A';
Creates a virtual table
Create View with Columns
CREATE VIEW student_grades (student_name, student_grade) AS SELECT name, grade FROM students;
View with custom column names
Select from View
SELECT * FROM top_students;
Queries view like a regular table
Update View
UPDATE top_students SET grade = 'A+' WHERE student_id = 101;
Updates data through view (with limitations)
Drop View
DROP VIEW top_students;
Removes a view
Materialized View
CREATE MATERIALIZED VIEW student_stats AS SELECT department, COUNT(*), AVG(age) FROM students GROUP BY department;
Stores query results physically (not all DBMS support)
Indexes Database objects that improve query performance
Create Index
CREATE INDEX idx_name ON students (name);
Improves query performance on name column
Unique Index
CREATE UNIQUE INDEX idx_email ON students (email);
Ensures unique values in column
Composite Index
CREATE INDEX idx_dept_grade ON students (department, grade);
Index on multiple columns
Drop Index
DROP INDEX idx_name ON students;
Removes an index
Clustered Index
CREATE CLUSTERED INDEX idx_id ON students (student_id);
Determines physical order of data (SQL Server syntax)
Stored Procedures Precompiled collection of SQL statements stored in database
Create Procedure
CREATE PROCEDURE get_student(IN sid INT) BEGIN SELECT * FROM students WHERE student_id = sid; END;
Creates a stored procedure
Call Procedure
CALL get_student(101);
Executes a stored procedure
Procedure with Parameters
CREATE PROCEDURE add_student( IN s_name VARCHAR(50), IN s_age INT, OUT new_id INT ) BEGIN INSERT INTO students (name, age) VALUES (s_name, s_age); SET new_id = LAST_INSERT_ID(); END;
Procedure with input and output parameters
Drop Procedure
DROP PROCEDURE get_student;
Removes a stored procedure
Functions Reusable routines that return a value
Create Function
CREATE FUNCTION get_student_count(dept VARCHAR(50)) RETURNS INT BEGIN DECLARE count INT; SELECT COUNT(*) INTO count FROM students WHERE department = dept; RETURN count; END;
Creates a user-defined function
Call Function
SELECT get_student_count('Computer');
Uses the function in a query
Drop Function
DROP FUNCTION get_student_count;
Removes a function
Triggers Automatic actions invoked by database events
Create Trigger
CREATE TRIGGER update_timestamp BEFORE UPDATE ON students FOR EACH ROW BEGIN SET NEW.updated_at = NOW(); END;
Automatically updates timestamp before update
Drop Trigger
DROP TRIGGER update_timestamp;
Removes a trigger
Transactions Groups of operations that execute as a single unit
Begin Transaction
BEGIN TRANSACTION;
Starts a transaction block
Commit
COMMIT;
Saves changes permanently
Rollback
ROLLBACK;
Undoes changes in transaction
Savepoint
SAVEPOINT point1;
Creates a point to rollback to
Rollback to Savepoint
ROLLBACK TO point1;
Rolls back to a specific savepoint
Transaction Isolation Levels
SET TRANSACTION ISOLATION LEVEL READ UNCOMMITTED | READ COMMITTED | REPEATABLE READ | SERIALIZABLE;
Controls transaction visibility to other transactions
Advanced Concepts Specialized SQL features and techniques
Common Table Expressions (CTE)
WITH dept_avg AS ( SELECT department, AVG(age) AS avg_age FROM students GROUP BY department ) SELECT * FROM dept_avg WHERE avg_age > 20;
Temporary result set for complex queries
Recursive CTE
WITH RECURSIVE hierarchy AS ( -- Base case SELECT id, name, manager_id FROM employees WHERE id = 101 UNION ALL -- Recursive case SELECT e.id, e.name, e.manager_id FROM employees e JOIN hierarchy h ON e.manager_id = h.id ) SELECT * FROM hierarchy;
Handles hierarchical or tree-structured data
Window Functions
SELECT name, grade, RANK() OVER (PARTITION BY department ORDER BY grade DESC) AS dept_rank FROM students;
Performs calculations across related rows
Pivot Data
SELECT * FROM ( SELECT department, grade, name FROM students ) AS source_table PIVOT ( COUNT(name) FOR grade IN ([A], [B], [C], [D], [F]) ) AS pivot_table;
Transforms rows into columns (syntax varies by DBMS)
Dynamic SQL
DECLARE @sql NVARCHAR(MAX); SET @sql = 'SELECT * FROM ' + @table_name; EXEC sp_executesql @sql;
Constructs and executes SQL dynamically (SQL Server syntax)
